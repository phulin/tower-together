# sim/ — Pure simulation core

No I/O, no Cloudflare dependencies, no Phaser. Fully unit-testable in Node.

## Files

### `index.ts`
`TowerSim` class — the public façade. Exposes `create()`, `from_snapshot()`, `step()`, `submit_command()`, `save_state()`, and read-only accessors. `step()` advances the clock by one tick and fires all checkpoints via `scheduler.ts`. `submit_command()` accepts sim-level commands only and delegates to `commands.ts`.

### `snapshot.ts`
Simulation snapshot helpers. Owns new-game snapshot creation plus snapshot normalization/migration for persisted saves before `TowerSim.from_snapshot()` reconstructs runtime state.

### `time.ts`
`TimeState` + `advanceOneTick()`. Tracks `day_tick` (0–2599), `daypart_index` (day_tick÷400), `day_counter`, `calendar_phase_flag`, `star_count`, and `total_ticks`. Constants: `DAY_TICK_MAX = 0x0a28`, `DAY_TICK_INCOME = 0x08fc`, `NEW_GAME_DAY_TICK = 0x9e5`. Two factory functions: `createTimeState()` (starts at tick 0, for unit tests) and `createNewGameTimeState()` (starts at 0x9e5 / daypart 6, matches the real game's `new_game_initializer`).

### `world.ts`
Grid constants (`GRID_WIDTH=375`, `GRID_HEIGHT=120`, lobby validation) and world data types. Key types: `PlacedObjectRecord` (18-byte record with spec-confirmed offsets; `stay_phase` at +0x0b, init values: `needs_refresh_flag=1`, `pairing_active_flag=1`, `pairing_status=-1`, `variant_index` = 1 for families 3/4/5/7/9/10 else 4), `GateFlags` + `createGateFlags()` (global star/security gate state, init per `new_game_initializer`), sidecar record types (`CommercialVenueRecord`, `ServiceRequestEntry`, `EntertainmentLinkRecord`), `CarrierRecord` (carrier_mode: 0=Express, 1=Standard, 2=Service elevator; escalators are NOT carriers), and the per-car / per-route runtime fields used for multi-car shaft assignment, explicit in-car route slots, route completion dispatch, and transfer-group/special-link rebuild state.

### `entities.ts`
Runtime entity population + Phase 4 behavior hooks. Rebuilds `world.entities` from placed objects (`rebuild_runtime_entities`), resets them at the end-of-day refresh checkpoints (`reset_entity_runtime_state`), advances the 1/16 refresh stride (`advance_entity_refresh_stride`), manages commercial venue counters/open-close state, computes simplified office/condo operational scoring, runs office-service checks, applies security/housekeeping tier updates, and refunds unhappy condos at rollover. Transport state now uses the recovered route-byte direction encoding more closely: carrier queue bytes use the upward/downward `0x40..` / `0x58..` bands, carrier completion dispatches family-visible arrival effects immediately during transport reconciliation, direct special-link legs are finalized on the per-tick reconciliation path only for actual transport transit states, and same-floor venue trips are treated as immediate-success arrivals instead of entering the transit queue path. For the current prototype, hotel room activation is no longer blocked by the unimplemented star-progression gate or the late new-game daypart, so hotel-heavy towers generate checkout demand immediately. `create_entity_state_records()` now includes live selected-floor plus carrier assignment/boarded metadata so the client can render riders queueing and moving with cars instead of pinning them to their home floors.

### `resources.ts`
All compile-time constants: `TILE_WIDTHS`, `TILE_COSTS`, `VALID_TILE_TYPES`, `FAMILY_CODE_TO_TILE` / `TILE_TO_FAMILY_CODE` (family ↔ tile-name mappings), `YEN_1001` (income payouts), `YEN_1002` (operating expenses), `OP_SCORE_THRESHOLDS`, `STAR_THRESHOLDS`, route delay constants. The overlay tile registry now includes `elevator`, `elevatorExpress`, and `elevatorService`, and YEN #1002 now exposes distinct standard / express / service elevator expense rates for carrier-side billing.

### `ledger.ts`
Three-ledger money model. `LedgerState` holds `cash_balance`, `primary_ledger[]`, `secondary_ledger[]`, `tertiary_ledger[]`, `cash_balance_cycle_base`. Key functions: `add_cashflow_from_family_resource()` (Phase 4 income), `rebuild_facility_ledger()` (called at 0x00f0), `do_expense_sweep()` (YEN #1002 charges), `do_ledger_rollover()` (3-day reset at 0x09e5).

### `scheduler.ts`
`SimState` bundle (`time + world + ledger`) and `run_checkpoints(state, prev_tick, curr_tick)`. Fires all 18 checkpoint bodies at the correct `day_tick` values: 0x000, 0x020, 0x0f0, 0x3e8, 0x4b0, 0x578, 0x5dc, 0x640, 0x6a4, 0x708, 0x76c, 0x7d0, 0x898, 0x8fc, 0x9c4, 0x9e5, 0x9f6, 0x0a06. Most Phase 3/4 bodies are stubs.

### `commands.ts`
`handle_place_tile()` and `handle_remove_tile()` — validate, mutate `WorldState + LedgerState`, create/free `PlacedObjectRecord` and sidecar records, call `run_global_rebuilds()` (facility ledger + carrier list + special links + walkability + transfer cache). Elevator segments are width-aware overlays and must align exactly with any adjacent shaft segment above/below; partially overlapping offsets are rejected. Overlay handling accepts `elevator`, `elevatorExpress`, and `elevatorService`, all sharing the same placement rules but rebuilding different carrier modes, and now rejects aligned shaft extensions whose adjacent segment uses a different mode. Also exports `SimCommand`, `CellPatch`, `CommandResult`, `fill_row_gaps()`.

### `carriers.ts`
`CarrierRecord` and `CarrierCar` state machine. `floor_to_slot(carrier, floor)` maps floor index to the carrier’s queueable/servable floor-slot model: modes 0/1 (local-mode) use up to 10 regular slots plus sky-lobby slots, while mode 2 (service/express-mode) uses direct offsets across the full span. `carrier_serves_floor(...)` is the shared source of truth used by both queueing and routing. `make_carrier(..., numCars?)` now supports up to 8 active cars with distributed home floors and initializes the recovered queue/slot structures: per-floor up/down 40-entry ring buffers, per-car 42 physical active route slots, destination counters, and per-shaft assignment capacity / waiting-car threshold fields. `rebuild_carrier_list(world)` scans only elevator-family overlays (`elevator`, `elevatorExpress`, `elevatorService`; escalators are special-link segments, not carriers), collapsing any overlay extension cells back onto their anchor column via `overlayToAnchor` so a 4-wide shaft rebuilds as one carrier. `tick_all_carriers(world, time)` uses daypart/calendar state for schedule gating, assigns floor requests to cars through the floor queues, drains queue entries into active slots at pickup floors, tracks authoritative destination counters, records completed route ids on unload, and drives the spec-shaped departure / dwell flow; carrier operating costs are charged elsewhere from `world.carriers`, not from placed-object records.

### `routing.ts`
Routing infrastructure. `rebuild_special_links(world)` now follows the recovered split: raw 64-entry special-link segments are built only from stairs/escalator overlays, while the 8 derived special-link records are synthesized around lobby / sky-lobby centers from walkability flags. `rebuild_walkability_flags(world)` is driven only by raw special-link segments. `rebuild_transfer_group_cache(world)` builds merged concourse clusters from family-24 objects using concourse footprint intersection against carrier columns, preserves tagged transfer entries, OR-projects a derived per-floor cache, and then rebuilds each derived special-link record's per-floor reachability table. `select_best_route_candidate(world, from, to, preferLocalMode?)` now uses the same carrier served-floor predicate as the queue layer, chooses a derived-record adjacent entry floor from the transfer-facing edge of the span rather than any arbitrary in-span transfer floor, and still keeps same-floor success in entity-side helpers rather than returning a synthetic route candidate.

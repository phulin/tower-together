# sim/ — Pure simulation core

No I/O, no Cloudflare dependencies, no Phaser. Fully unit-testable in Node.

## Files

### `index.ts`
`TowerSim` class — the public façade. Exposes `create()`, `fromSnapshot()`, `step()`, `submitCommand()`, `saveState()`, and read-only accessors. `step()` advances the clock by one tick and fires all checkpoints via `scheduler.ts`. `submitCommand()` accepts sim-level commands only and delegates to `commands.ts`. Snapshot hydration/defaulting and persistence cloning now live in `snapshot.ts`, keeping the class focused on runtime orchestration plus query helpers.

### `snapshot.ts`
Simulation snapshot helpers. Owns new-game snapshot creation plus snapshot normalization/migration for persisted saves, runtime hydration/defaulting (`hydrateSnapshot()`), and snapshot-safe cloning of live sim state for persistence (`serializeSimState()`).

### `time.ts`
`TimeState` + `advanceOneTick()`. Tracks `day_tick` (0–2599), `daypart_index` (day_tick÷400), `day_counter`, `calendar_phase_flag`, `star_count`, and `total_ticks`. Constants: `DAY_TICK_MAX = 0x0a28`, `DAY_TICK_INCOME = 0x08fc`, `NEW_GAME_DAY_TICK = 0x9e5`. Two factory functions: `createTimeState()` (starts at tick 0, for unit tests) and `createNewGameTimeState()` (starts at 0x9e5 / daypart 6, matches the real game's `new_game_initializer`).

### `world.ts`
Grid constants (`GRID_WIDTH=375`, `GRID_HEIGHT=120`, lobby validation) and world data types. Key types: `PlacedObjectRecord` (18-byte record with spec-confirmed offsets; `unitStatus` at +0x0b, `evalActiveFlag` at +0x14, `evalLevel` at +0x15, `rentLevel` at +0x16), `GateFlags` + `createGateFlags()` (global star/progression state plus recycling-center adequacy, the daily family-345 sale counter, and the newspaper latch), sidecar record types (`CommercialVenueRecord`, `ServiceRequestEntry` with optional `floorIndex`/`coverageFlag` for parking demand, `EntertainmentLinkRecord` with selector/phase/budget/attendance fields), `parkingDemandLog` (sidecar indices of uncovered parking spaces), `SimNotification` / `SimPrompt` (pending news, event, and prompt outputs), `EventState` (bomb/fire state, deterministic bomb-search cursor, carrier-edit prompt target), `CarrierRecord` (carrier_mode: 0=Express, 1=Standard, 2=Service elevator; escalators are NOT carriers), and the per-car / per-route runtime fields used for multi-car shaft assignment, explicit in-car route slots, route completion dispatch, and transfer-group/special-link rebuild state.

### `entities.ts`
Runtime entity population + Phase 4 behavior hooks. Rebuilds `world.entities` from placed objects (`rebuildRuntimeEntities`), resets them at the end-of-day refresh checkpoints (`resetEntityRuntimeState`), advances the 1/16 refresh stride (`advanceEntityRefreshStride`), manages commercial venue counters/open-close state, computes shared hotel/office/condo operational scoring with updated support-family and radius rules (including bidirectional `evalActiveFlag` and same-floor pairing for offices), runs the office-service evaluation gate from the office entity state machine, applies recycling-center tier updates with the recovered no-scale / tier-5 preservation rules, refunds unhappy condos only on the shared 3-day cashflow pass, and synchronously cleans up entity route state and carrier pending routes on tile demolition (`cleanupEntitiesForRemovedTile`). Condo sales now happen through the outbound commercial-trip reservation path instead of the old lobby-to-home shortcut, while condo venue selection still branches on `baseOffset % 4` to yield restaurant / fast-food / fast-food across the 3-occupant span. Parking demand: `rebuildParkingDemandLog` populates `world.parkingDemandLog` from uncovered parking service-request sidecars; `tryAssignParkingService` random-picks from the log for hotel suite and office consumers gated by star level and eligibility checks. Office entities now use a spec-faithful daily state machine: 0x27 (parked) → 0x20 (morning activation with daypart/calendar gating) → 0x00 (commute to office) → 0x21 (at office) → 0x01 (venue visit) → 0x05 (evening departure) → 0x27; cashflow remains on the 3-day cadence via `auxValueOrTimer` guard. Venue visits now enter an explicit dwell state keyed off `word0a` instead of bouncing back immediately on arrival. Hotel checkout now passes through the recovered sentinel/countdown rewrite path and updates the daily family-345 sale counter / newspaper latch on realized payouts. Transport state uses the recovered route-byte direction encoding: carrier queue bytes use the upward/downward `0x40..` / `0x58..` bands, carrier completion dispatches family-visible arrival effects immediately during transport reconciliation, direct special-link legs are finalized on the per-tick reconciliation path only for actual transport transit states, and same-floor venue trips still resolve as immediate arrivals into the dwell pipeline instead of entering a carrier queue. Entertainment venue records now use a single `linkPhaseState` plus selector-driven budget seeding, age saturation, and the corrected cinema nonzero-attendance payout rule. For the current prototype, hotel room activation is no longer blocked by the unimplemented star-progression gate or the late new-game daypart, so hotel-heavy towers generate checkout demand immediately. `createEntityStateRecords()` now includes live selected-floor plus carrier assignment/boarded metadata so the client can render riders queueing and moving with cars instead of pinning them to their home floors.

### `entertainment.ts`
Paired (cinema) and single (entertainment) link state machines. Owns budget seeding, selector-driven age tiers, forward/reverse half activation, phase advance, attendance-tiered cinema payouts, and reset-to-parked logic for entertainment entities. Called from the scheduler's entertainment checkpoints.

### `cathedral.ts`
Cathedral evaluation entities (families 0x24–0x28). Handles the day-start activation gate, morning dispatch to the eval zone, midday return routing, and the all-arrived award/tower-promotion path. Called from the scheduler's start-of-day and hotel-sale checkpoints, and per-entity from the state dispatcher in `entities.ts`.

### `resources.ts`
All compile-time constants: `TILE_WIDTHS`, `TILE_COSTS`, `VALID_TILE_TYPES`, `FAMILY_CODE_TO_TILE` / `TILE_TO_FAMILY_CODE` (family ↔ tile-name mappings), `YEN_1001` (income payouts), `YEN_1002` (operating expenses), `OP_SCORE_THRESHOLDS`, `STAR_THRESHOLDS`, route delay constants (`DELAY_STOP_ESCALATOR` / `DELAY_STOP_STAIRS` for Escalator-branch and Stairs-branch per-stop delays). The overlay tile registry now includes `elevator`, `elevatorExpress`, and `elevatorService`, and YEN #1002 now exposes distinct standard / express / service elevator expense rates for carrier-side billing.

### `ledger.ts`
Three-ledger economy model. `LedgerState` holds `cashBalance`, `populationLedger[]`, `incomeLedger[]`, `expenseLedger[]`, `cashBalanceCycleBase`. Key functions: `addCashflowFromFamilyResource()` (Phase 4 income), `rebuildFacilityLedger()` (called at 0x00f0), `doExpenseSweep(ledger, world, starCount)` (YEN #1002 charges for facilities, carriers, stair/escalator raw links, and star-dependent parking expenses via `PARKING_EXPENSE_RATE_BY_STAR`), `doLedgerRollover()` (3-day cycle-base snapshot + ledger reset before the fresh expense pass at 0x09e5).

### `scheduler.ts`
`SimState` bundle (`time + world + ledger`) and `runCheckpoints(state, prev_tick, curr_tick)`. Fires all 18 checkpoint bodies at the correct `day_tick` values: 0x000, 0x020, 0x0f0, 0x3e8, 0x4b0, 0x578, 0x5dc, 0x640, 0x6a4, 0x708, 0x76c, 0x7d0, 0x898, 0x8fc, 0x9c4, 0x9e5, 0x9f6, 0x0a06. All checkpoint bodies are implemented: daily fire/bomb triggering now runs at 0x0f0 in recovered order (fire before bomb), entertainment phase advance at 0x5dc (paired forward) and 0x640 (reverse + accrue income), afternoon/end-of-day notifications, carrier end-of-day flush at 0x9f6, entertainment budget seeding at 0x0f0, entertainment entity activation at 0x3e8/0x4b0/0x578, and the condo refund pass now only runs on the shared 3-day rollover cadence.

### `commands.ts`
`handlePlaceTile()` and `handleRemoveTile()` — validate, mutate `WorldState + LedgerState`, create/free `PlacedObjectRecord` and sidecar records, call `runGlobalRebuilds()` (facility ledger + entities + parking demand log + carrier list + special links + walkability + transfer cache). Recycling-center placement enforces a service-stack overlap constraint: subsequent 0x14/0x15 slices must be within `[existingFloor - 2, existingFloor + 1]` of an existing stack, and legacy `security` / `housekeeping` command IDs are normalized to the new recycling-center upper/lower tile names. Parking tiles now receive a `ServiceRequestEntry` sidecar with `floorIndex` for demand-log participation. Tile demolition synchronously cleans up entity route state via `cleanupEntitiesForRemovedTile` before the global rebuild. Elevator segments are width-aware overlays and must align exactly with any adjacent shaft segment above/below; partially overlapping offsets are rejected. Overlay handling accepts `elevator`, `elevatorExpress`, and `elevatorService`, all sharing the same placement rules but rebuilding different carrier modes, and now rejects aligned shaft extensions whose adjacent segment uses a different mode. Carrier-car removal now emits a `carrier_edit_confirmation` prompt when live shaft traffic would be disrupted, with immediate removal only on the no-traffic path or after prompt acceptance. Also exports `SimCommand`, `CellPatch`, `CommandResult`, `fillRowGaps()`.

### `ring-buffer.ts`
Generic fixed-capacity `RingBuffer<T>` class. Backed by a pre-allocated array with `push`, `pop`, `peekAll`, `size`, `isFull`, `isEmpty`. Serializes as plain JSON and reconstitutes via `RingBuffer.from()`. Used by carrier floor queues.

### `carriers.ts`
`CarrierRecord` and `CarrierCar` state machine. `floorToSlot(carrier, floor)` maps floor index to the carrier’s queueable/servable floor-slot model: modes 0/1 (local-mode) use up to 10 regular slots plus sky-lobby slots, while mode 2 (service/express-mode) uses direct offsets across the full span. `carrierServesFloor(...)` is the shared source of truth used by both queueing and routing. `makeCarrier(..., numCars?)` now supports up to 8 active cars with distributed home floors and initializes the recovered queue/slot structures: per-floor up/down 40-entry ring buffers, per-car 42 physical active route slots, destination counters, and per-shaft assignment capacity / waiting-car threshold fields. `rebuildCarrierList(world)` scans only elevator-family overlays (`elevator`, `elevatorExpress`, `elevatorService`; escalators are special-link segments, not carriers), collapsing any overlay extension cells back onto their anchor column via `overlayToAnchor` so a 4-wide shaft rebuilds as one carrier. `tickAllCarriers(world, time)` uses daypart/calendar state for schedule gating, assigns floor requests to cars through the floor queues, drains queue entries into active slots at pickup floors, tracks authoritative destination counters, records completed route ids on unload, and drives the spec-shaped departure / dwell flow; carrier operating costs are charged elsewhere from `world.carriers`, not from placed-object records.

### `events.ts`
Bomb, fire, random-news, and VIP special visitor event systems. `tryTriggerBombEvent()` fires at `day_counter % 60 == 59` for stars 2–4: selects a random floor/tile, arms a bomb with `BOMB_DEADLINE_TICKS` (1200) deadline, and seeds the deterministic bomb-search cursor when emergency-response coverage exists. `tickBombEvent()` runs per-tick, advancing the search cursor before deadline resolution; `checkSecurityPatrolBomb()` resolves if the scan reaches the hidden bomb tile. Blast area is 6 floors × 40 tiles centered on bomb position. `tryTriggerFireEvent()` fires at `day_counter % 84 == 83` for stars 3+: two-front per-floor fire spread with configurable rates, helicopter rescue/extinguish. `tickFireEvent()` advances spread and checks resolution. `triggerRandomNewsEvent()` runs in the pre-checkpoint per-tick hook with the recovered RNG gate ordering. `tickVipSpecialVisitor()` now uses metro placement/floor state and toggles the metro object's display aux word instead of legacy hotel VIP flags. `handlePromptResponse()` now also resolves deferred carrier-edit confirmations. `checkDailyEvents()` dispatches fire-before-bomb at the 0x0f0 checkpoint.

### `routing.ts`
Routing infrastructure. `rebuildSpecialLinks(world)` now follows the recovered split: raw 64-entry special-link segments are built only from stairs/escalator overlays (stairs overlay → Stairs branch with stairs cost bit set; escalator overlay → Escalator branch with stairs cost bit clear), while the 8 derived special-link records are synthesized around lobby / sky-lobby centers from walkability flags. `rebuildWalkabilityFlags(world)` is driven only by raw special-link segments (bit 0 = Escalator-branch route support, bit 1 = Stairs-branch route support). `rebuildTransferGroupCache(world)` builds merged concourse clusters from family-24 objects using concourse footprint intersection against carrier columns, preserves tagged transfer entries, OR-projects a derived per-floor cache, and then rebuilds each derived special-link record's per-floor reachability table. `selectBestRouteCandidate(world, from, to, preferLocalMode?)` now uses the same carrier served-floor predicate as the queue layer, chooses a derived-record adjacent entry floor from the transfer-facing edge of the span rather than any arbitrary in-span transfer floor, and still keeps same-floor success in entity-side helpers rather than returning a synthetic route candidate.

# apps/client/src/game

Phaser 3 game rendering layer.

- **GameScene.ts** — Phaser `Scene` subclass. Renders the tower grid with tall cells (`4 px` wide × `16 px` tall) so facilities display at the intended `4:1` height:width ratio. Uses shared `TILE_WIDTHS` for hover/placement previews, merged floor/lobby row rendering, and multi-cell facility drawing; the standard insertion cursor is drawn as one rectangle over the full placement footprint instead of per-cell boxes, and elevator/escalator shaft overlays are merged into contiguous vertical outlines per aligned shaft. Draws centered single-character labels on anchor tiles for major room/facility types (`O`, `R/T/S`, `C`, etc.) so the in-world plan is readable at a glance. Also draws runtime people/entities as small rectangles in a dedicated overlay layer, colored by stress (`black`/`pink`/`red`), but only while they are actively queueing for elevator travel; idle/home occupants and boarded riders are hidden, while pre-board trip state `0x22` is still shown in the queue so active demand remains visible before assignment. Renders elevator cars moving inside shafts from carrier-state wire updates and overlays each car with a live boarded-occupancy number derived from boarded entity state. Multi-car shafts are laid out from `carIndex`/`carCount` in the wire state so the client remains responsible for car placement inside the shaft, and the scene now keeps timestamped entity/carrier snapshots plus a presentation clock so dynamic overlays redraw every frame with interpolation/prediction between less-frequent authoritative updates. Camera setup now clamps both startup zoom and wheel/pinch zoom to a shared `1x..4x` range, and wheel/pinch anchoring uses Phaser world-point transforms so zoom stays centered on the cursor instead of drifting vertically. Exposes `applyInitState(cells, simTime, entities, carriers)`, `applyPatch(cells)`, `applyEntities(simTime, entities)`, `applyCarriers(simTime, carriers)`, `setPresentationClock(...)`, and `setOnCellClick(handler)` so `GameScreen` can inject the click dispatcher.
- **PhaserGame.tsx** — React component that creates and destroys the `Phaser.Game` instance. Receives `onCellClick` and `sceneRef` as props; wires the click callback into the scene whenever it changes.
- **gameSceneConstants.ts** — Shared visual/layout constants extracted from `GameScene`, including tile dimensions, colors, label registries, zoom bounds, and family sizing tables used by render helpers. Service-stack visuals now label the paired recycling-center slices rather than the older security/housekeeping names.
- **gameScenePlacement.ts** — Pure placement helpers extracted from `GameScene`, including shift-fill computation and hover-preview bounds for the currently selected tool.
- **gameSceneTransport.ts** — Pure transport/presentation helpers extracted from `GameScene`, including snapshot timing, car interpolation, queue positioning, and car bounding-box calculations.
- **transportSelectors.ts** — Shared transport selectors and counters consumed by both the React HUD and Phaser scene. Keeps queued-entity detection, car occupancy aggregation, moving-car detection, and debug metrics consistent across client surfaces.

# apps/client/src/game

Phaser 3 game rendering layer.

- **GameScene.ts** — Phaser `Scene` subclass. Renders the tower grid with tall cells (`4 px` wide × `16 px` tall) so facilities display at the intended `4:1` height:width ratio. Uses shared `TILE_WIDTHS` for hover/placement previews, merged floor/lobby row rendering, and multi-cell facility drawing; the standard insertion cursor is drawn as one rectangle over the full placement footprint instead of per-cell boxes, and elevator/escalator shaft overlays are merged into contiguous vertical outlines per aligned shaft. Draws centered single-character labels on anchor tiles for major room/facility types (`O`, `R/T/S`, `C`, etc.) so the in-world plan is readable at a glance. Also draws runtime people/entities as small rectangles in a dedicated overlay layer, colored by stress (`black`/`pink`/`red`), but only while they are actively queueing for elevator travel; idle/home occupants and boarded riders are hidden. Renders elevator cars moving inside shafts from carrier-state wire updates and overlays each car with a live boarded-occupancy number derived from boarded entity state. Multi-car shafts are laid out from `carIndex`/`carCount` in the wire state so the client remains responsible for car placement inside the shaft. Supports camera pan (middle/right mouse drag or two-finger swipe) and zoom (scroll wheel, pinch, or `Shift` + two-finger swipe). Exposes `applyInitState(cells, entities, carriers)`, `applyPatch(cells)`, `applyEntities(entities)`, `applyCarriers(carriers)`, and `setOnCellClick(handler)` so `GameScreen` can inject the click dispatcher.
- **PhaserGame.tsx** — React component that creates and destroys the `Phaser.Game` instance. Receives `onCellClick` and `sceneRef` as props; wires the click callback into the scene whenever it changes.
